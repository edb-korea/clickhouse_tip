# ClickHouse에서 테이블/MV 등 오브젝트 확인 방법

ClickHouse는 메타데이터를 `system` 데이터베이스의 시스템 테이블로 제공하기 때문에, 표준 SQL로 대부분의 오브젝트 정보를 조회할 수 있습니다.

## 1. 테이블 목록 확인

### 기본 SHOW 명령어
```sql
SHOW TABLES;                     -- 현재 데이터베이스의 테이블
SHOW TABLES FROM my_database;    -- 특정 DB 지정
```

### system.tables로 더 상세히 조회 (권장)
```sql
SELECT database, name, engine, total_rows, total_bytes, create_table_query
FROM system.tables
WHERE database = 'my_database'
ORDER BY name;
```

- `engine` 컬럼으로 MergeTree/Log/Distributed 등 엔진 종류 확인 가능
- `total_rows`, `total_bytes`로 크기 파악 가능
- `create_table_query`에 전체 DDL이 그대로 들어있어 `SHOW CREATE TABLE` 없이도 확인 가능

## 2. Materialized View(MV) 목록만 따로 확인

MV도 `system.tables`에 들어있고, 이름 관례가 아니라 **`engine` 컬럼으로 정확히 구분**합니다.

```sql
-- 방법 1: system.tables의 engine 컬럼으로 필터 (권장)
SELECT database, name, engine, as_select
FROM system.tables
WHERE engine = 'MaterializedView';
```

- `as_select` 컬럼에 MV 생성 시 사용한 원본 SELECT 쿼리가 그대로 보입니다.

```sql
-- 방법 2: 이름 패턴으로도 확인 가능 (관례상 _mv 접미사를 쓰는 경우)
SELECT database, name
FROM system.tables
WHERE name LIKE '%_mv';
```

방법 1이 훨씬 정확합니다 (이름 관례에 의존하지 않음).

## 3. 데이터베이스 목록

```sql
SHOW DATABASES;

-- 더 상세히
SELECT name, engine, data_path
FROM system.databases;
```

## 4. 뷰(일반 View, MV 아님) 목록

```sql
SELECT database, name, as_select
FROM system.tables
WHERE engine = 'View';
```

## 5. Dictionary(딕셔너리) 목록

```sql
SHOW DICTIONARIES;

-- 상세
SELECT database, name, type, status, element_count
FROM system.dictionaries;
```

## 6. 컬럼 정보(특정 테이블의 스키마) 확인

```sql
DESCRIBE TABLE my_database.my_table;

-- 또는 system.columns로 여러 테이블 한 번에
SELECT database, table, name, type, default_expression
FROM system.columns
WHERE database = 'my_database' AND table = 'my_table';
```

## 7. 특정 테이블의 원본 DDL(CREATE 문) 확인

```sql
SHOW CREATE TABLE my_database.my_table;
```

## 8. 전체 오브젝트를 한 번에 종합적으로 파악하고 싶을 때

```sql
SELECT
    database,
    name,
    engine,
    CASE
        WHEN engine = 'MaterializedView' THEN 'Materialized View'
        WHEN engine = 'View' THEN 'View'
        WHEN engine LIKE '%MergeTree%' THEN 'Table (MergeTree family)'
        WHEN engine = 'Distributed' THEN 'Distributed Table'
        WHEN engine IN ('Kafka', 'PostgreSQL', 'MySQL', 'S3') THEN 'Integration Table'
        ELSE engine
    END AS object_type,
    total_rows,
    total_bytes
FROM system.tables
WHERE database NOT IN ('system', 'information_schema', 'INFORMATION_SCHEMA')
ORDER BY database, object_type, name;
```

이 쿼리 하나로 **테이블 / MV / 일반 View / Distributed / Integration 엔진 테이블**을 유형별로 한눈에 구분해서 볼 수 있습니다.

## 9. MV가 데이터를 채워주는 대상 테이블까지 함께 추적

MV는 보통 별도의 저장 테이블(`TO` 절로 지정하거나, 내부적으로 자동 생성된 `.inner` 테이블)에 데이터를 씁니다. 이 연결관계까지 보려면:

```sql
SELECT database, name, as_select
FROM system.tables
WHERE engine = 'MaterializedView';
```

`as_select` 안의 `FROM` 절로 원본 소스 테이블을 확인할 수 있고, 데이터가 실제로 쌓이는 대상 테이블까지 정확히 보려면:

```sql
SELECT name, create_table_query
FROM system.tables
WHERE engine = 'MaterializedView';
```

`create_table_query`에 `CREATE MATERIALIZED VIEW ... TO target_table AS SELECT ...` 형태로 전체 정의가 나옵니다.

## 참고 — `information_schema`도 지원

MySQL/PostgreSQL 호환 도구 연동을 위해 표준 `information_schema.tables`, `information_schema.columns`도 지원하지만, ClickHouse 고유 정보(엔진, 압축, 파티션 등)는 `system.*` 테이블이 훨씬 풍부하니 실무에서는 `system.tables`를 기본으로 쓰시는 걸 추천드립니다.
